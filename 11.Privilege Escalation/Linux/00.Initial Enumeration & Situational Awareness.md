# Initial Enumeration & Situational Awareness
**Concept:** Upon gaining initial shell access, establishing situational awareness is critical. This phase maps the operating system, network configuration, user context, and running services to identify misconfigurations or legacy components vulnerable to privilege escalation.
## 1. System & Environment Information
**Goal:** Identify the OS distribution, kernel version, architecture, and current environment variables. This dictates payload compatibility (x86 vs x64) and kernel exploit viability.

Retrieve OS release information and Kernel details. 
```shell
cat /etc/os-release ; uname -a
```

Inspect CPU architecture to determine compatible exploit binaries. 
```shell
lscpu
```

Review current environment variables and execution PATH. Writable paths can be abused for PATH hijacking. 
```shell
env ; echo $PATH
```

List available shell interpreters on the system. 
```shell
cat /etc/shells
```
## 2. User & Group Enumeration
**Goal:** Map the current user context, identify other users, and locate high-value groups (e.g., `sudo`, `docker`, `adm`).

**Description:** Extract all users from the passwd file, filtering for those with valid login shells. 
```shell
grep "*sh$" /etc/passwd
```

**Description:** List all groups and verify members of the `sudo` group. 
```shell
#Syntax: cat /etc/group ; getent group <Group_Name>
cat /etc/group ; getent group sudo
```

**Description:** Identify currently logged-in users and recent login history. 
```shell
w ; lastlog
```

**Description:** Enumerate user command history files to hunt for hardcoded credentials. 
```shell
find / -type f \( -name *_hist -o -name *_history \) -exec ls -l {} \; 2>/dev/null
```
**OPSEC Warning:** Running `find` from the root directory (`/`) generates significant disk I/O and can trigger Endpoint Detection and Response (EDR) or File Integrity Monitoring (FIM) alerts.
## 3. Network & Routing
**Goal:** Discover internal subnets, dual-homed interfaces, and neighboring hosts for lateral movement.

Check connections.
```shell
conncheck
```

Enumerate network interfaces, IP addresses, and the routing table. 
```shell
ip a ; route
```

Check the ARP cache for recently contacted hosts and local DNS entries. 
```shell
arp -a ; cat /etc/hosts
```
## 4. File System & Storage
**Goal:** Identify mounted/unmounted drives, hidden files, and temporary directories that often house scripts, backups, or credentials.

**Description:** List block devices and currently mounted file systems. 
```shell
lsblk ; df -h
```

**Description:** Parse `/etc/fstab` to identify file systems configured to mount on boot (including unmounted ones). 
```shell
cat /etc/fstab | grep -v "#" | column -t
```

**Description:** Search for hidden files and directories across the file system. 
```shell
# Syntax: find <Path> -type <f|d> -name ".*" -ls 2>/dev/null`
# Example (Files):
find / -type f -name ".*" -exec ls -l {} \; 2>/dev/null | grep htb-student

# Example (Directories): 
find / -type d -name ".*" -ls 2>/dev/null
```

**Description:** Inspect temporary directories for leftover artifacts, scripts, or session files. 
```shell
ls -la /tmp /var/tmp /dev/shm
```
## 5. Services, Cron Jobs & Processes
**Goal:** Identify processes running as root, scheduled tasks, and vulnerable installed packages.

List all running processes, filtering for those executing as `root`. 
```shell
# Syntax: ps aux | grep <User>
ps aux | grep root
```

Check for daily scheduled cron jobs. 
```shell
ls -la /etc/cron.daily/
```

Extract command-line arguments used to launch running processes from the `proc` file system (often reveals passwords passed as arguments). 
```shell
find /proc -name cmdline -exec cat {} \; 2>/dev/null | tr "\0" "\n"
```

Search for configuration files that may contain hardcoded credentials or service parameters. 
```shell
find / -type f \( -name *.conf -o -name *.config \) -exec ls -l {} \; 2>/dev/null
```
## 6. Vulnerable Binaries & Software
**Goal:** Find SUID binaries, misconfigured system tools, or outdated packages.

Check the `sudo` version for known vulnerabilities (e.g., CVE-2021-3156 "Baron Samedit"). 
```shell
sudo -V
```

Intercept and record system calls made by a binary (useful for finding missing shared objects or configuration files). 
```shell
# Syntax: strace <Command>
strace ping -c1 10.10.10.10
```

Cross-reference locally installed packages with the GTFOBins database to identify binaries that can be abused to bypass local security restrictions. 
```shell
# Syntax: apt list --installed | tr "/" " " | cut -d" " -f1,3 | sed 's/[0-9]://g' | tee -a installed_pkgs.list
for i in $(curl -s https://gtfobins.github.io/ | html2text | cut -d" " -f1 | sed '/^[[:space:]]*$/d'); do
    if grep -q "$i" installed_pkgs.list; then
        echo "Check GTFO for: $i"
    fi
done
```

**OPSEC Warning:** The `curl` command inside the loop makes an outbound HTTP connection directly from the compromised target to `gtfobins.github.io`. This is highly anomalous for a production server and will likely trigger network IDS/IPS or firewall alerts. _Tradecraft recommendation:_ Export the `installed_pkgs.list` to your attack machine and run the comparison offline.
